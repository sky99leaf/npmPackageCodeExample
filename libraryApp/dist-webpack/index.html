This is index.html in dist-webpack
Run with either:

  • node_modules/webpack/bin/webpack serve -c ./webpack.config_devServer.js and browse to localhost:3000
  • node_modules/webpack/bin/webpack build -c ./webpack.config_devServer.js
    to build dev bundle dist-webpack/bundle_localdev.js without running server, then directly clicking dist-webpack/index.html

How this page gets javascript and React.js from the bundle

  • This html page has <script src="./bundle_localdev.js"...
  • Files in the bundle get visible from DOM object window.MyLibrary.
  • A subsequent script tag calls putStuffToCodeInDiv() from the bundle.
  • This function does document.getElementById('divThatGetsReactContent') that's in this html file
    and renders a React functional component into it. That component should have components to be coded and tested so you can see the results here
  • when you run index.html without a server, it shows that everything is in plain javascript, including React.js functionality.
    That's because babel "transpiled" the JSX in source code to plain javascript, gave it to webpack which included it in bundle_localdev.js
  • file webpack.config_devServer.js runs the show in this process
